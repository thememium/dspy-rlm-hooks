# Reporting a Vulnerability

To report a security vulnerability, please email boswell.labs@gmail.com.

We take security seriously and will respond to security reports within 48 hours. Please include as much detail as possible about the vulnerability, including:

- A description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Suggested fix (if any)

While the discovery of new vulnerabilities is rare, we also recommend always using the latest version of dspy-rlm-hooks to ensure your application remains as secure as possible.

## Security Considerations for dspy-rlm-hooks

As dspy-rlm-hooks instruments the internal iteration loop of DSPy's RLM, which executes LLM-generated Python code, please be aware of the following security practices:

- **Monkey-Patching**: This package modifies private DSPy internals (`_execute_iteration`, `_aexecute_iteration`, etc.) at runtime. Future DSPy releases may change these internals, which could lead to unexpected behaviour or security gaps. Always pin to a known-compatible DSPy version.
- **Code Execution**: The underlying RLM executes arbitrary Python code generated by a language model. Hooks do not prevent unsafe code from running — they only observe or rewrite it. Use `pre_execution` hooks to sandbox or reject dangerous operations, but do not rely on hooks alone for security.
- **Hook Injection**: Any code with access to an RLM instance can call `enable_rlm_hooks()` and inject arbitrary behaviour into the iteration loop. Ensure you trust the source of any hooks you install.
